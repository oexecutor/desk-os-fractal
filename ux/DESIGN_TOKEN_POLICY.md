# Política de tokens de design

A identidade visual é compartilhada entre tela e impressão, mas acessibilidade pode exigir variantes semânticas.

- `#14140f`, `#1f4b3f`, `#b3792c`, `#a8492f` e fundos principais são identidade normativa.
- Em tela, texto secundário deve usar variante com contraste WCAG suficiente; `ink-faint` original pode ser restrito a decoração não essencial.
- Na impressão, metadados técnicos podem usar tom mais claro desde que não sejam a única forma de comunicar informação.
- Estados sempre combinam cor, texto e/ou símbolo.
- Tokens de componente não devem usar cores literais fora do package de tokens.
