# Glossário

| Termo | Definição normativa |
|---|---|
| Workspace | Limite lógico que agrupa projetos, configuração, eventos e permissões. Não é mecanismo de autenticação. |
| Portfólio | Raiz opcional que agrupa múltiplos projetos. |
| Projeto | Unidade de resultado com objetivo, restrições, árvore e ciclo de vida. |
| Nó | Unidade recursiva canônica da árvore. |
| Bloco | Nó operacional que contém preferencialmente três ações atômicas. |
| Ação atômica | Menor unidade executável, verificável e atribuível. |
| LINK | Entregável-síntese produzido pelo fechamento de um bloco. Não é uma quarta ação manual. |
| Fractal | Navegação por zoom na mesma árvore, com mudança de escala e contexto. |
| Projeção | Visão derivada do modelo canônico, sem duplicar a fonte de verdade. |
| Sprint | Projeção temporal de nós ativos em uma janela de execução. |
| Emitir | Gerar snapshot físico imprimível a partir do estado atual aprovado. |
| Face 1 | Dashboard físico de visão, orientação e estado. |
| Face 2 | Dashboard físico de foco, ação e fechamento. |
| Recycle | Decisão ao final do bloco: continuar, reduzir, dividir ou reconfigurar. |
| QR contextual | QR que abre exatamente um contexto ou comando registrado no router. |
| Plano gerado | Estrutura criada pelo motor, ainda sem autorização para execução. |
| Plano ativo | Plano validado e aprovado por humano, apto para execução e impressão. |
| Evidência | Referência ou artefato que demonstra conclusão ou origem de informação. |
| Snapshot | Estado materializado em um instante, usado por UI, sincronização e impressão. |
